## Урок 01: Линейная алгебра — интуиция (Linear Algebra Intuition)

### Векторы — это точки (и направления)

Вектор — список чисел. Числа = координаты в пространстве.

```
v = [3, 2]  — 2D вектор, длина √(9+4) = √13
```

В AI векторы представляют всё:
- Слово → вектор из 768 чисел (эмбеддинг)
- Изображение → вектор из миллионов пикселей
- Пользователь → вектор предпочтений

### Матрицы — это преобразования

Матрица преобразует один вектор в другой: поворачивает, масштабирует, растягивает, проецирует.

```
Нейросеть: веса = матрицы, которые преобразуют вход в выход
Attention = матрица, решает, на что обращать внимание
Эмбеддинги = матрица, отображает слова в векторы
```

### Скалярное произведение (Dot Product) = мера схожести

```
a · b = a₁×b₁ + a₂×b₂ + ... + aₙ×bₙ

Одно направление:  a · b > 0  (похожи)
Перпендикулярны:   a · b = 0  (не связаны)
Противоположные:   a · b < 0  (различны)
```

Это то, как работают поиск, рекомендации и RAG — ищем векторы с высоким dot product.

### Линейная независимость

Набор векторов линейно независим, если ни один нельзя выразить через другие.

```
v1 = [1, 0, 0]
v2 = [0, 1, 0]
v3 = [2, 1, 0]   — v3 = 2*v1 + v2 → зависимы!
```

В ML: если два признака идеально коррелированы, модель не может разграничить их влияние → мультиколлинеарность.

### Ранг матрицы

Ранг = количество линейно независимых столбцов (или строк).

| Ситуация | Ранг | Значение для ML |
|---|---|---|
| Полный ранг | min(m,n) | Уникальное решение, модель стабильна |
| Дефектный ранг | < min(m,n) | Признаки избыточны, нужны регуляризация |

### Проекция

Проекция **a** на **b** — компонента **a** в направлении **b**:

```
proj_b(a) = (a·b / b·b) × b
```

- Линейная регрессия = проекция на столбцовую пространство
- PCA = проекция на направления максимальной дисперсии
- Attention в трансформерах = проекция запросов на ключи

### Процесс Грамма-Шмидта

Преобразование любого набора векторов в **ортонормальный базис** (все единичной длины, все перпендикулярны).

```
u1 = v1 / |v1|
u2 = (v2 - proj_u1(v2)) / |...|
u3 = (v3 - proj_u1(v3) - proj_u2(v3)) / |...|
```

Это QR-разложение: Q = ортонормальный базис, R = коэффициенты проекций.

### Связь с AI

| Концепция | Где применяется |
|---|---|
| Dot product | Attention scores, cosine similarity в RAG |
| Матричное умножение | Каждый слой нейросети |
| Линейная независимость | Отбор признаков, мультиколлинеарность |
| Ранг | LoRA (адаптация с низким рангом) |
| Проекция | Линейная регрессия, PCA |
| QR-разложение | Численные методы, вычисление собственных значений |

## Урок 02: Векторы, матрицы и операции (Vectors, Matrices & Operations)

### Векторы

```
v = [3, 4]      — 2D, длина 5 (3-4-5 треугольник)
w = [1, 0, -2]  — 3D
```

### Матрицы

```
A = | 1  2  3 |   — 2×3 матрица (2 строки, 3 столбца)
    | 4  5  6 |
```

### Формула умножения

```
(m × n) @ (n × p) = (m × p)

(128 × 784) @ (784 × 1) = (128 × 1)
  веса        вход        выход
```

Внутренние размерности **должны совпадать**. Ошибка形状 в PyTorch = именно это.

### Операции

| Операция | Что делает | Где в нейросети |
|---|---|---|
| Сложение | Поэлементное | Добавление bias |
| Скалярное умножение | Масштабирование | lr × градиенты |
| Матричное умножение | Преобразование векторов | Forward pass слоя |
| Транспонирование | Строки ↔ столбцы | Backpropagation |
| Определитель | Одно число | Проверка обратимости |
| Обратная матрица | Отмена преобразования | Решение линейных систем |
| Единичная матрица | Ничего не делает | Residual connections |

### Поэлементное vs матричное умножение

**Поэлементное** (одинаковая форма):
```
| 1  2 |   | 5  6 |   | 5  12 |
| 3  4 | * | 7  8 | = | 21 32 |
```

**Матричное** (внутренние размерности совпадают):
```
| 1  2 |   | 5  6 |   | 19  22 |
| 3  4 | @ | 7  8 | = | 43  50 |
```

### Broadcasting

Когда добавляешь bias к матрице, формы не совпадают. Broadcasting растягивает меньший массив:

```
| 1  2  3 |   +   [10, 20, 30]
| 4  5  6 |

→  | 1  2  3 |   | 10  20  30 |   | 11  22  33 |
   | 4  5  6 | + | 10  20  30 | = | 14  25  36 |
```

### Нейросеть = матричные операции

```
output = relu(W @ x + b)
```

Один слой = одно матричное умножение + сложение bias + активация.

## Урок 03: Матричные преобразования (Matrix Transformations)

### Матрица = машина变形ирования пространства

| Преобразование | Матрица (2D) | Что делает |
|---|---|---|
| **Поворот** | `[[cos θ, -sin θ], [sin θ, cos θ]]` | Вращает точки по дуге окружности, сохраняя расстояния |
| **Масштаб** | `[[sx, 0], [0, sy]]` | Растягивает/сжимает по осям |
| **Сдвиг (shear)** | `[[1, kx], [ky, 1]]` | Прямоугольники → параллелограммы |
| **Отражение** | `[[-1, 0], [0, 1]]` | Зеркально отражает через ось |

### 3D повороты

```
Rz(θ): вращение в плоскости xy (z остаётся)
Rx(θ): вращение в плоскости yz (x остаётся)
Ry(θ): вращение в плоскости xz (y остаётся)
```

### Композиция: порядок важен

```
Применить A потом B = B @ A @ точка
Матричное умножение НЕ коммутативно: B @ A ≠ A @ B
```

### Собственные числа и векторы (Eigen)

```
A @ v = λ * v

v  — собственный вектор (направление, которое матрица НЕ поворачивает)
λ  — собственное число (во сколько раз растягивает)
```

Зачем:
- **PCA** — собственные векторы ковариационной матрицы = главные компоненты
- **Устойчивость RNN** — |λ| > 1 → взрыв градиентов, |λ| < 1 → затухание
- **Спектральная кластеризация** — собственные числа матрицы смежности

### Определитель = масштаб объёма

```
det = 1   → область сохранена (поворот)
det = 2   → область удвоена
det = 0   → вырожденная матрица (пространство схлопнулось)
det = -1  → область сохранена, ориентация перевёрнута (отражение)
```

## Урок 04: Calculus for ML

### Производная = наклон касательной

```
f'(x) = lim(h→0) [f(x+h) - f(x)] / h

f(x) = x²  →  f'(x) = 2x
f(x) = eˣ  →  f'(x) = eˣ
f(x) = ln(x) → f'(x) = 1/x
```

### Числовая производная (аппроксимация)

```
f'(x) ≈ [f(x+h) - f(x-h)] / (2h)    h ≈ 1e-7
```

### Частные производные и градиент

```
f(x, y) = x² + 3xy + y²
df/dx = 2x + 3y   (y — константа)
df/dy = 3x + 2y   (x — константа)

grad f = [df/dx, df/dy]  — вектор самого крутого подъёма
```

### Градиентный спуск

```
w_new = w_old - learning_rate * dL/dw

learning rate слишком большой → расходимся
learning rate слишком маленький → медленная сходимость
```

### Цепное правило (Chain Rule)

```
y = f(g(x))  →  dy/dx = f'(g(x)) * g'(x)

Нейросеть: input → linear → activation → linear → loss
Backprop = цепное правило, применённое справа налево
```

### Гессиан (матрица вторых производных)

```
H[i][j] = d²f / (dx_i · dx_j)

H положительно определённая (λ > 0) → локальный минимум (чаша)
H отрицательно определённая (λ < 0) → локальный максимум
H неопределённая (смешанные λ) → седловая точка
```

### Методы оптимизации

| Метод | Использует | Стоимость | Сходимость |
|---|---|---|---|
| Градиентный спуск | 1-е производные | O(N) | Медленная |
| Метод Ньютона | Полный гессиан | O(N³) | Быстрая |
| L-BFGS | Аппроксимация гессиана | O(N) | Средняя |
| Adam | Адаптивные скорости | O(N) | Средняя |

### Ряд Тейлора

```
f(x+h) ≈ f(x) + f'(x)·h + ½f''(x)·h² + ...

1-й порядок → градиентный спуск
2-й порядок → метод Ньютона
```

## Урок 05: Chain Rule & Automatic Differentiation

### Автоматическое дифференцирование

Три компонента:
1. **Value** — обёртка над числом (значение + градиент)
2. **Graph recording** — каждая операция записывает входы и backward-функцию
3. **Backward** — топологическая сортировка → обратный обход → chain rule

### Forward vs Reverse Mode

| Mode | Направление | Когда лучше |
|---|---|---|
| **Forward** | Входы → Выход | Мало входов, много выходов |
| **Reverse** | Выход → Вход | Много входов, мало выходов (нейросети!) |

Reverse mode = backpropagation = **один** обратный проход для всех градиентов.

### Нейросеть (micrograd)

```
Neuron:   tanh(w1*x1 + w2*x2 + ... + b)
Layer:    [Neuron, Neuron, ...]
MLP:      [Layer, Layer, ...]
```

Каждый вес — это `Value`. Вызываешь `loss.backward()` → градиенты текут к каждому параметру.

### Gradient checking

```
autograd vs числовая производная
Разница должна быть < 1e-5
```

## Урок 06: Probability & Distributions

### Аксиомы вероятности

1. P(A) ≥ 0
2. P(S) = 1 (что-то обязательно произойдёт)
3. P(A or B) = P(A) + P(B) если A и B несовместны

### Условная вероятность

```
P(A|B) = P(A and B) / P(B)

Независимость: P(A and B) = P(A) * P(B)
```

### Распределения

| Распределение | Формула | Где в ML |
|---|---|---|
| **Bernoulli** | P(X=1)=p, P(X=0)=1-p | Бинарная классификация |
| **Categorical** | P(X=i)=pᵢ, Σpᵢ=1 | Мультикласс (softmax) |
| **Normal** | f(x) = exp(-(x-μ)²/2σ²) / √(2πσ²) | Веса, шум, градиенты |
| **Poisson** | P(X=k) = λᵏe⁻λ/k! | Частота событий |
| **Uniform** | f(x) = 1/(b-a) | Инициализация весов |

### Мат. ожидание и дисперсия

```
E[X] = Σ xᵢ · P(X=xᵢ)     — среднее
Var(X) = E[(X - E[X])²]     — разброс
σ = √Var(X)                 — стандартное отклонение
```

### Правило 3 сигм (Normal)

```
68% данных в ±1σ
95% данных в ±2σ
99.7% данных в ±3σ
```

### Центральная предельная теорема (ЦПТ)

Среднее многих независимых величин → **нормальное распределение**. Поэтому Гаусс везде в ML.

### Softmax

```
softmax(zᵢ) = exp(zᵢ) / Σexp(zⱼ)

Хитрость: вычесть max(z) перед exp() — предотвращает overflow
```

### Log-вероятности

```
log(a * b) = log(a) + log(b)

Произведение P → underflow после ~30 множителей
log P → сумма, численно стабильно
```

### Cross-entropy loss

```
L = -log_softmax(logits)[target]

Это минус логарифм вероятности правильного класса.
Чем меньше — тем лучше модель.
```

## Урок 07: Bayes' Theorem — байесово рассуждение

### Формула Байеса

```
P(A|B) = P(B|A) × P(A) / P(B)

Постериор = (Правдоподобие × Априори) / Доказательство
```

| Часть | Название | Смысл |
|---|---|---|
| P(A\|B) | Постериор | Обновлённая вероятность A после увиденного B |
| P(B\|A) | Правдоподобие | Насколько вероятно B, если A истинно |
| P(A) | Априори | Вероятность A до данных |
| P(B) | Доказательство | Общая вероятность B |

### Пример: медицинский тест

```
Болезнь: 1 из 10 000 → P(sick) = 0.0001
Тест точен на 99% → P(+|sick) = 0.99
Ложный позитив: 1% → P(+|healthy) = 0.01

P(sick|+) = 0.99 × 0.0001 / (0.99×0.0001 + 0.01×0.9999)
          = 0.0098 = 0.98%

→ Менее 1%! Априори доминирует.
```

### Наивный Байес — классификатор

Предположение: признаки независимы при данном классе.

```
score(class) = P(class) × ∏ P(feature_i | class)

Логарифмируем:
log score = log P(class) + Σ log P(feature_i | class)
```

**Сглаживание Лапласа** — защита от нулевых вероятностей:
```
P(word|class) = (count + 1) / (total + vocab_size)
```

### MLE vs MAP

| Оценка | Оптимизирует | ML-аналог |
|---|---|---|
| **MLE** | P(data\|params) | Без регуляризации |
| **MAP** | P(data\|params) × P(params) | L2/L1 регуляризация |

Гауссов априори = L2. Лапласов априори = L1.

### Байесово обновление (Beta-Binomial)

```
Prior:     Beta(a, b)
Данные:    s успехов, f неудач
Posterior: Beta(a + s, b + f)

Сегодняшний постерior = завтрашний априори
```

### A/B тестирование

```
Вариант A: 50/1000 → Beta(51, 951)
Вариант B: 65/1000 → Beta(66, 936)

P(B > A) = доля сэмплов, где B > A
> 0.95 → запускаем B
```

### Связь с ML

- **Априори = регуляризация** — Gaussian prior = L2
- **Постериор = неуверенность** — распределение, а не точка
- **Байесово обновление = онлайн-обучение** — постерior → априори

## Урок 08: Optimization — семейство градиентного спуска

### Градиентный спуск (vanilla)

```
w = w - lr × gradient
```

Learning rate — самый важный гиперпараметр:
- Слишком большой → расходимся
- Слишком маленький → ползём тысячами шагов

### SGD vs Batch vs Mini-batch

| Вариант | Batch | Градиент | Скорость | Шум |
|---|---|---|---|---|
| Batch GD | Весь датасет | Точный | Медленно | Нет |
| SGD | 1 сэмпл | Очень шумный | Быстро | Высокий |
| Mini-batch | 32-256 | Хорошая оценка | Баланс | Умеренный |

Шум помогает выходить из седловых точек.

### Momentum — мяч, катящийся вниз

```
v = beta × v + gradient
w = w - lr × v
```

Накапливает скорость, гасит колебания. beta=0.9 — дефолт.

### Adam — адаптивные learning rates

```
m = β₁ × m + (1-β₁) × g          (1-й момент — среднее градиентов)
v = β₂ × v + (1-β₂) × g²         (2-й момент — среднее квадратов)

m̂ = m / (1 - β₁ᵗ)                 (bias correction)
v̂ = v / (1 - β₂ᵗ)

w = w - lr × m̂ / (√v̂ + ε)
```

Деление на √v̂: веса с большими градиентами → маленький шаг, с малыми → большой.

Дефолты: `lr=0.001, β₁=0.9, β₂=0.999, ε=1e-8`.

### Learning rate schedules

| Расписание | Формула | Когда |
|---|---|---|
| Step decay | lr × factor каждые N эпох | Простое управление |
| Exponential | lr₀ × decay^t | Плавное снижение |
| Cosine | lr_min + ½(lr_max-lr_min)(1+cos(πt/T)) | Трансформеры |
| Warmup + decay | Линейный рост → decay | Крупные модели |

### Выпуклые vs невыпуклые

- **Выпуклая** → один минимум, GD всегда находит
- **Невыпуклая** (нейросети) → много локальных минимумов, седловые точки

Седловые точки — главная проблема. Momentum и mini-batch шум помогают покинуть.

## Урок 09: Information Theory — энтропия, cross-entropy, KL

### Информационное содержание (Surprise)

```
I(x) = -log(p(x))

Событие              Вероятность    Удивление (биты)
Монетка орёл         0.5            1.0
Выпало 6             0.167          2.58
1 из 1000            0.001          9.97
Наверняка            1.0            0.0
```

### Энтропия (среднее удивление)

```
H(P) = -Σ p(x) · log(p(x))

Честная монета: H = 1.0 бит (максимум)
Смещённая (99%): H = 0.08 бит
```

Энтропия = неустранимая неуверенность. Нельзя сжать ниже.

### Перекрёстная энтропия (Cross-Entropy)

```
H(P, Q) = -Σ p(x) · log(q(x))

P — истинное распределение (метки)
Q — предсказания модели
```

Для классификации (one-hot):
```
H(P, Q) = -log(q(правильный класс))
```

Это `CrossEntropyLoss`. Минимизируя её, увеличиваем P(правильный класс).

### KL-расхождение

```
D_KL(P || Q) = H(P, Q) - H(P) = Σ p(x) · log(p(x)/q(x))

CE = энтропия + KL
```

Минимизировать CE = минимизировать KL. Толкаем модель к истинному распределению.

**KL не симметрично**: D_KL(P||Q) ≠ D_KL(Q||P)

### Взаимная информация (Mutual Information)

```
I(X; Y) = H(X) - H(X|Y) = H(X) + H(Y) - H(X,Y)

I = 0 → X и Y независимы
I = H(X) → X полностью определяет Y
```

Для отбора признаков: высокая MI = признак полезен. Correlation ловит только линейные связи, MI — все.

### Label smoothing

```
soft_target = (1-ε) * hard_target + ε / num_classes

Hard: [0, 0, 1, 0]  → энтропия = 0
Soft: [0.025, 0.025, 0.925, 0.025]  → энтропия = 0.5

Увеличивает энтропию целей → регуляризация, калибровка
```

### Перплексия (Perplexity)

```
Perplexity = 2^H(P,Q)    (биты)
Perplexity = e^H(P,Q)    (наты)

Модель с perplexity 50 "сбита с толку" как при выборе из 50 вариантов.
GPT-2: ~30, современные модели: единицы.
```

### Почему CE — лучшая loss

```
1. Теория информации:CE = среднее удивление (минимизируем waste bits)
2. Максимальное правдоподобие: CE = NLL (максимизируем P(data))
3. Градиент: dL/dz_i = softmax(z_i) - one_hot(i) — чистый, стабильный
```

## Урок 10: Dimensionality Reduction — PCA, t-SNE, UMAP

### Проклятие размерности

```
Размерность    Отношение расстояний (max/min)
2              ~5.0
10             ~1.8
100            ~1.2
1000           ~1.02

→ Расстояния бессмысленны, нужно экспоненциально больше данных
```

### PCA (Principal Component Analysis)

```
1. Центрируем (вычитаем среднее)
2. Ковариационная матрица
3. Собственная декомпозиция
4. Сортируем по собственным значениям (больше дисперсии → раньше)
5. Проекция на top k компонент
```

**Explained variance ratio** — какую долю дисперсии сохраняет каждая компонента.

**Выбор k:**
- Порог: 90-95% кумулятивной дисперсии
- Elbow method: где кривая "ломается"
- Downstream: какая точность модели лучше

### t-SNE

- Нелинейный, для **визуализации** в 2D
- Сохраняет **локальных соседей**
- Стохастический, медленный O(n²)
- Не для препроцессинга!

### UMAP

- Быстрее t-SNE
- Сохраняет **локальную + глобальную** структуру
- Параметры: `n_neighbors`, `min_dist`

### Когда что

| Метод | Зачем | Сохраняет | Скорость |
|---|---|---|---|
| **PCA** | Препроцессинг, сжатие | Глобальную дисперсию | Быстро |
| **t-SNE** | Визуализация 2D | Локальных соседей | Медленно |
| **UMAP** | Визуализация + структура | Локальную + глобальную | Средне |
| **Kernel PCA** | Нелинейные данные | Сложные структуры | Средне |

### Kernel PCA

Kernel trick для PCA: проецирует в пространство высокой размерности.

| Kernel | Формула | Для чего |
|---|---|---|
| RBF | exp(-γ‖x-y‖²) | Гладкие многообразия |
| Polynomial | (x·y+c)^d | Полиномиальные связи |

## Урок 11: SVD — Singular Value Decomposition

### Формула SVD

```
A = U × Σ × Vᵀ

U:     m×m, ортогональная (левые сингулярные векторы)
Σ:     m×n, диагональная (сингулярные значения ≥ 0)
Vᵀ:    n×n, ортогональная (правые сингулярные векторы)
```

### Геометрический смысл

```
A × vᵢ = σᵢ × uᵢ

Матрица A берёт vᵢ, масштабирует на σᵢ, отображает в uᵢ
```

### Truncated SVD — лучшее приближение ранга k

**Теорема Эккарта-Youngа**: обрезка по top-k → оптимальное приближение.

```
Aₖ = Uₖ × Σₖ × Vₖᵀ
Ошибка = σₖ₊₁ + ... + σᵣ
```

### Применения

| Применение | Как |
|---|---|
| Сжатие изображений | Урезаем мелкие σ |
| Рекомендации | Латентные факторы (Netflix Prize) |
| NLP: LSA | Сингулярные векторы = темы |
| Шумоподавление | Сигнал в top-k, шум в остальных |
| Псевдообратная | Решение переопределённых систем |

### PCA = SVD

```
X = U × Σ × Vᵀ
Principal components = правые сингулярные V
Объяснённая дисперсия = σᵢ² / (n-1)
```

### Псевдообратная

```
A = U Σ Vᵀ  →  A⁺ = V Σ⁺ Uᵀ

Σ⁺: инвертируем ненулевые σᵢ, остальные = 0
Решение: x = A⁺ b (minimizes ||Ax - b||)
```

## Урок 12: Tensor Operations — тензоры и einsum

### Что такое тензор

```
Скаляр:  rank 0, shape ()
Вектор:  rank 1, shape (3,)
Матрица: rank 2, shape (2,3)
4D:      (B, C, H, W) — изображения (PyTorch)
         (B, T, D)    — NLP
         (B, H, T, D) — attention
```

### Broadcasting

```
A: (8, 1, 6, 1)
B:    (7, 1, 5)
→ padded B: (1, 7, 1, 5)
→ Result:   (8, 7, 6, 5)

Правила: выравнивание справа, dim = или dim = 1
```

### Einsum

```
"i,i->"         — dot product
"i,j->ij"       — outer product
"ik,kj->ij"     — matmul
"bij,bjk->bik"  — batch matmul
"bhtd,bhsd->bhts" — attention scores
"ii->"          — trace
"ij->ji"        — transpose
```

### Multi-head Attention

```
Input:        (B, T, E)
Q = X @ W_q: (B, T, E) → reshape → (B, H, T, D)
scores:       (B, H, T, T)  ← attention map
weights:      softmax(scores)
output:       (B, H, T, D) → merge → (B, T, E)
```

### Каждый слой = tensor операция

| Слой | Формула | Einsum |
|---|---|---|
| Linear | Y = X @ W.T + b | "bd,od->bo" |
| Attention Q | Q = X @ W_q | "btd,dh->bth" |
| Attention scores | Q @ K.T / √d | "bhtd,bhsd->bhts" |
| Attention output | softmax @ V | "bhts,bhsd->bhtd" |

## Урок 13: Numerical Stability — численная стабильность

### IEEE 754

```
float32: ~7 знаков, до 3.4e38
float16: ~3 знака, до 65 504     ← мало для ML!
bfloat16: ~2 знака, до 3.4e38    ← range как float32
```

### Overflow и Underflow

```
exp(88.7) = 3.4e38  (предел float32)
exp(89.0) = inf     ← overflow!
exp(-104) = 0.0     ← underflow!
log(0.0)  = -inf
```

### Log-Sum-Exp Trick

```
log(sum(exp(x_i))) = max(x) + log(sum(exp(x_i - max(x))))

Без трюка: exp(100) = inf
С трюком:  exp(100-102) = exp(-2) = 0.135
```

### Stable Softmax

```
max_logit = max(logits)
exps = [exp(z - max_logit) for z in logits]
```

### Gradient Checking

```
relative_error = |analytical - numerical| / max(|analytical|, |numerical|)

< 1e-7: отлично
< 1e-5: нормально
> 1e-3: ошибка
```

### Mixed Precision

```
float16: быстрый, limited range → нужен loss scaling
bfloat16: быстрый + широкий range → без loss scaling
```

## Урок 14: Norms and Distances — нормы и расстояния

### Нормы

| Норма | Формула | Когда |
|---|---|---|
| **L1** | Σ\|xᵢ\| | Разреженные данные, устойчивость к выбросам |
| **L2** | √(Σxᵢ²) | Евклидова, пространственные данные |
| **L∞** | max(\|xᵢ\|) | Worst-case deviation |
| **Cosine** | a·b / (\|\|a\|\| × \|\|b\|\|) | Текст, эмбеддинги (направление > длина) |

### Расстояния

| Расстояние | Для чего |
|---|---|
| L1/L2 | Векторы, пространственные данные |
| Cosine | Текст, эмбеддинги, рекомендации |
| Mahalanobis | Выбросы, коррелированные признаки |
| Jaccard | Множества (теги, категории) |
| Edit | Строки (опечатки, ДНК) |
| KL | Распределения (не симметрично!) |
| Wasserstein | GAN, распределения без пересечения |

### Связь с регуляризацией

```
L1 (Lasso): loss + λ||w||₁ → разреженные веса
L2 (Ridge): loss + λ||w||₂² → мелкие веса
```

## Урок 15: Statistics for ML — статистика для ML

### Описательная статистика

```
Среднее: μ = Σxᵢ/n — точка баланса
Медиана: середина отсортированных — устойчива к выбросам
σ: √дисперсия — разброс в исходных единицах
IQR: Q3-Q1 — разброс середины 50%
```

### Корреляция

| Тип | Что ловит | Когда |
|---|---|---|
| **Pearson** | Линейную связь | Нормальные данные, нет выбросов |
| **Spearman** | Монотонную связь | Ранги, выбросы, ненормальные данные |

### Гипотеза и p-value

```
H0: разницы нет (дефолт)
H1: разница есть

p-value = P(данные такие или более экстремальные | H0 верна)
p < 0.05 → отвергаем H0
```

### Доверительные интервалы

```
95% CI = x̄ ± 1.96 × (s/√n)
```

### Bootstrap

```
1. n сэмплов С ВОЗВРАТОМ
2. Считаем статистику
3. Повторяем B раз
4. 95% CI = [2.5%, 97.5%] перцентили
```

### Размер эффекта (Cohen's d)

```
d = (mean₁ - mean₂) / pooled_std

d = 0.2: маленький, 0.5: средний, 0.8: большой

Всегда: p-value + effect size
```

## Урок 16: Sampling Methods — методы семплирования

### Методы семплирования

| Метод | Как | Когда |
|---|---|---|
| Inverse CDF | X = F⁻¹(U) | Есть замкнутая форма CDF |
| Rejection | Предлагаем + принимаем/отвергаем | Есть PDF до константы |
| Importance | Взвешиваем сэмплы из q(x) | Нужен E_p[f(x)] |
| Monte Carlo | Среднее сэмплов | Интегралы без замкнутой формы |

### Генерация текста (LLM)

| Метод | Как | Когда |
|---|---|---|
| Temperature | Делим logits на T | T<1 → уверенно, T>1 → креативно |
| Top-k | Оставляем k лучших | Фиксированный набор |
| Top-p | Оставляем набор с суммой ≥ p | Адаптивный набор |

### Reparameterization Trick

```
z ~ N(μ, σ²)  → не дифференцируемо
z = μ + σ × ε  → дифференцируемо! (ε ~ N(0,1))
```

### MCMC

```
Metropolis-Hastings: предлагаем x', принимаем min(1, p(x')/p(x))
Gibbs: обновляем одну переменную из условного распределения
```

## Урок 17: Linear Systems — линейные системы

### Методы решения Ax = b

| Метод | Требования | Стоимость | Когда |
|---|---|---|---|
| Gaussian elimination | Квадратная A | O(n³) | Разовое решение |
| LU | Квадратная A | O(n³) + O(n²) | Множество решений |

| Normal equations | m > n | O(mn²+n³) | Линейная регрессия |

### Normal Equations = Linear Regression

```
A^T A x = A^T b
w = (X^T X)^(-1) X^T y

Ridge: w = (X^T X + λI)^(-1) X^T y
```

### Condition Number

```
κ = σ_max / σ_min

κ < 100: безопасно
κ ~ 10^k: теряем k знаков
κ ~ 10^16: ответ бессмысленный
```

## Урок 18: Convex Optimization — выпуклая оптимизация

### Выпуклые функции

```
f(tx + (1-t)y) ≤ t·f(x) + (1-t)·f(y)

→ Локальный минимум = глобальный минимум
→ GD всегда находит оптимум
```

### Метод Ньютона

```
GD:      x_new = x - lr × grad
Ньютон:  x_new = x - H⁻¹ × grad

Квадратичная сходимость, но O(n²) памяти
```

### KKT условия

```
1. Стационарность
2. Первичная допустимость: gᵢ(x) ≤ 0
3. Двойственная допустимость: λᵢ ≥ 0
4. Комплементарная нежёсткость: λᵢ·gᵢ(x) = 0
```

### Регуляризация = ограничённая оптимизация

```
L2 (Ridge): min Loss(w) s.t. ||w||² ≤ t  → круг → веса мелкие
L1 (LASSO): min Loss(w) s.t. ||w||₁ ≤ t  → ромб → спарсивность
```

## Урок 19: Complex Numbers — комплексные числа

### Основы

```
z = a + bi,  i² = -1

|z| = √(a² + b²)  — модуль
arg(z) = atan2(b, a) — фаза

```
e^(iθ) = cos θ + i sin θ

e^(iπ) + 1 = 0  ←最美的 формула

```
z × e^(iθ) = поворот на угол θ

Это ТО ЖЕ, что матрица поворота:
[[cos θ, -sin θ], [sin θ, cos θ]]
```

### DFT

```
X[k] = Σ x[n] × e^(-2πikn/N)

Корни единицы: w_k = e^(2πik/N) — N точек на единичной окружности
```

### Связь с Transformer

```
Sinusoidal PE: sin(pos/10000^(2i/d)), cos(pos/10000^(2i/d))
→ real и imag части e^(iθ)

RoPE: q × e^(iθ) → относительная позиция = угол поворота
```
